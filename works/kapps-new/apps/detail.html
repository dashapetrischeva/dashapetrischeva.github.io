FIPO Helpdesk

Web-basert helpdesk

Beskrivelse

Noen funksjoner

  • O365 innlogging
  • Mobiltilpasset
  • Meld inn saker via brukerportal eller e-post
  • Sett opp flere helpdesker i samme installasjon, med egne saksnummer (f.eks. forskjellige avdelinger)
  • Innebygd nettbutikk, som også lar deg knytte varer og forbruk til en sak
  • Faktura-rapport
  • Kategorisering som tjeneste, cmdb, kategori, lokasjon…
  • Automatisk tildeling av saker basert på lokasjon eller kategori.
  • Redigerbare e-postmaler
  • API for automatisering
  • Mulighet for automatisk lukking av saker basert på tid

Installasjonsveiledning

Pr. dags dato ligger kode i et privat BitbBucket repository. For hver gang det publiseres endringer i GIT, så lages det automatisk en nytt docker-image, (som forøvrig også er privat). Målet er å få disse over til public, men har valgt privat under selve utviklingen.

Om du ønsker tilgang for å teste, så ta gjerne kontakt med oss.

Vi kjører testmiljø i Docker på en Ubuntu-server og produksjonsmiljø kjøres i Kubernetes på Digital Ocean.

Under er noen av konfigurasjonsfilene, men disse bør tilpasses ditt eget miljø. F.eks. er det ikke sikkert du har behov for å kjøre database og phpmyadmin i containere.

Innlogging

Innlogging har fokus på Office365 og er fremhevet. Men støttes også innlogging med brukernavn/passord.

Så pr. dags dato anbefales helpdesk sammen med Office365.

Her må det da settes opp en Enterprise applikasjon i ditt miljø, med Applikasjonstilgang for User.Read.All. Nøkler legges som variabel i docker-oppsett.

Docker


## docker-compose.yaml
## FIPO Helpdesk

version: '3.2'
services:

  ## API
  fipo-frontend:
    container_name: fipo-frontend
    image: fosenikt/fipo-frontend:latest
    restart: always
    ports:
      - "8090:80"
    volumes:
      - "/etc/timezone:/etc/timezone:ro"
      - "/etc/localtime:/etc/localtime:ro"
    environment:
        - VIRTUAL_HOST=fipo.fikt.no
        - API_URL=fipoapi.fikt.no
    networks:
      - default

  ## API
  fipo-api:
    container_name: fipo-api
    image: fosenikt/fipo-backend:latest
    restart: always
    ports:
      - "8091:80"
    volumes:
      - fipo-upload:/var/www/html/upload
      - "/etc/timezone:/etc/timezone:ro"
      - "/etc/localtime:/etc/localtime:ro"
    environment:
        - VIRTUAL_HOST=fipoapi.mittdomene.no
        - FRONTEND_HOST=fipo.mittdomene.no
        - DB_NAME=fipo
        - DB_USER=fipo
        - DB_PASSWORD=Kake123Passord
        - DB_HOST=fipo-db
        - MEMCACHED_HOST=fipo-memcached
        - MEMCACHED_PORT=11211
        - O365_TENANT_ID=TENANT_ID
        - O365_APP_ID=APP_ID
        - O365_APP_SECRET=APP_SECRET
        - JWT_ISSUER=TOKEN_ISSUER
        - JWT_SECRET=TOKEN_SECRET
        - API_SOURCE_DIR=/var/www/html/
        - UPLOAD_FOLDER=/upload
        - IPGEOLOCATION_API_KEY=API_KEY
    networks:
      - default


  fipo-schedule:
    build:
      context: ./fipo-schedule/
      dockerfile: .docker/Dockerfile
    restart: always
    container_name: fipo_schedule
    ports:
      - "8081:9005"
    volumes:
      - type: bind
        source: /var/www/fip-online-app/fipo-schedule/src
        target: /var/www/html
        bind:
          propagation: shared
      - "/etc/timezone:/etc/timezone:ro"
      - "/etc/localtime:/etc/localtime:ro"
    environment:
      - CRON_USER=root
      - CRON_SCHEDULE=*/1 * * * *
      - CRON_COMMAND=php /var/www/html/run.php
      - APACHE_RUN_USER=www-data
      - APACHE_RUN_GROUP=www-data
      - API_URL=fipo-api
      - TOKEN="TOKEN_FRA_API"
    networks:
      - default


  ## Memcached
  fipo-memcached:
    container_name: fipo-memcached
    image: memcached:latest
    restart: always
    ports:
        - "11212:11211"
    networks:
      - default


  ## Database
  fipo-db:
    container_name: fipo-db
    build:
      context: ./fipo-db/
      dockerfile: .docker/Dockerfile
    restart: always
    command: --default-authentication-plugin=mysql_native_password
    restart: always
    environment:
      MYSQL_DATABASE: 'fipo_helpdesk'
      # So you don't have to use root, but you can if you like
      MYSQL_USER: 'fipo_db_user'
      # You can use whatever password you like
      MYSQL_PASSWORD: 'Kake123Passord'
      # Password for root access
      MYSQL_ROOT_PASSWORD: 'Kake123Passord'
      TZ: Europe/Oslo
    ports:
      #  : < MySQL Port running inside container>
      - '3306:3306'
      # Where our data will be persisted
    volumes:
      - ./fipo-db:/docker-entrypoint-initdb.d
      - fipo-db:/var/lib/mysql
      - "/etc/timezone:/etc/timezone:ro"
      - "/etc/localtime:/etc/localtime:ro"

  ## PHPmyadmin
  fipo-phpmyadmin:
    container_name: fipo_phpmyadmin
    image: phpmyadmin/phpmyadmin
    links:
      - fipo-db
    environment:
      VIRTUAL_HOST: fipdb.fikt.no
      PMA_HOST: fipo-db
      PMA_PORT: 3306
    ports:
      - '8080:80'
    volumes: 
      - /sessions

## VOLUMES
volumes:
    persistent:
    fipo-db:
    fipo-upload:

networks:
  default:
    external:
      name: webproxy
Kubernetes
## fipo-helpdesk-k8-backend-deployment.yaml
## FIPO Helpdesk Backend (API)

apiVersion: apps/v1
kind: Deployment
metadata:
  name: fipo-backend-deployment
  labels:
    app: fipo-backend
spec:
  replicas: 2
  selector:
    matchLabels:
      app: fipo-backend
  template:
    metadata:
      labels:
        app: fipo-backend
    spec:
      containers:
       - name: fipo-backend
         image: fosenikt/fipo-backend:latest
         imagePullPolicy: Always
         ports:
         - containerPort: 80
         env:
          - name: DB_NAME
            value: fipo_helpdesk
          - name: DB_USER
            value: fipo_helpdesk
          - name: DB_PASSWORD
            value: PASSORD
          - name: DB_HOST
            value: DATABASE_SERVER
          - name: DB_PORT
            value: '3306'
          - name: O365_APP_ID
            value: APP_ID
          - name: O365_APP_SECRET
            value: 'APP_SECRET'
          - name: JWT_ISSUER
            value: TOKEN_ISSUER
          - name: JWT_SECRET
            value: TOKEN_SECRET
          - name: URL_BACKEND
            value: //helpdesk-api.mittdomene.no/
          - name: URL_FRONTEND
            value: //helpdesk.mittdomene.no/
      imagePullSecrets:
       - name: dockerhub-secret
